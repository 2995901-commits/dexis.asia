## 3. Граф источников

### 3.1. Визуальная концепция

Force-directed graph в стиле Obsidian Graph View (граф связей с «физикой» — узлы взаимно притягиваются и отталкиваются, можно перемещать мышью).

**В центре:** круглый узел DEXIS (диаметр 96px). Изумрудный заливочный цвет (#10b981 или близкий из брендбука). Тонкая светящаяся обводка.

**Вокруг — 7 основных узлов источников:**

1. **1С** — при наведении показать перечень конфигураций (Бухгалтерия, Управление торговлей, УНФ, Комплексная автоматизация)
2. **ИС ЭСФ** — информационная система электронных счетов-фактур РК
3. **Налоговые системы РК** — СОНО и КГД (объединённый узел)
4. **CRM** — Bitrix24, amoCRM
5. **Банк-клиенты** — Halyk, Каспи Business, Forte
6. **Таблицы** — Excel, Google Sheets
7. **Госзакупки** — goszakup.gov.kz

**Серые узлы «в разработке» (показываются по клику на кнопку «Показать все источники»):** Stat.gov.kz, Национальный Банк РК, WhatsApp Business, ERP-системы (SAP, 1С:ERP).

### 3.2. Поведение

- Линии между узлами и DEXIS — тонкие, полупрозрачные (opacity 30%)
- При наведении на узел источника:
  - Узел увеличивается в 1.1 раза
  - Линия к DEXIS становится ярче (opacity 100%)
  - Появляется tooltip-карточка с описанием: «Подтягивает: остатки на счетах, выручку по направлениям, маржу, дебиторку, кредиторку»
- Узлы можно перетаскивать мышью — они возвращаются на место после отпускания
- На мобильных устройствах: вместо drag — статичное расположение в радиальной сетке

### 3.3. Технологии

- SVG + D3.js (JavaScript-библиотека для визуализации данных) для force-directed layout
- Или Cytoscape.js (альтернативная библиотека для графов)
- Без анимации частиц на этом этапе — это статичный граф

## 4. Motion-нарратив

### 4.1. Общие параметры

- **Длительность:** 30 секунд
- **Размещение:** под графом источников, на той же ширине, в выделенной зоне (затемнённый фон с тонкой рамкой)
- **Запуск:** по кнопке «Посмотреть, как это работает» (manual trigger)
- **Альтернативный запуск:** scroll-trigger при попадании раздела в viewport (по умолчанию выключен, чтобы не отвлекать)
- **Возможность перезапуска:** кнопка «Посмотреть ещё раз» появляется после завершения
- **Звук:** отсутствует. Опциональная кнопка «Голос-сопровождение основателя» — на будущее, не в первой версии

### 4.2. Технологии

- GSAP (GreenSock Animation Platform — JavaScript-библиотека для motion design)
- SVG для всех графических элементов
- ScrollTrigger plugin от GSAP — если будет включён автозапуск
- MotionPath plugin — для движения точек по линиям

### 4.3. АКТ 1 — Хаос данных (секунды 0–9)

**Layout:** условная сетка 12 столбцов × 8 рядов.

**Появление карточек источников (последовательно):**

| Время | Источник | Положение | Содержимое карточки |
|---|---|---|---|
| t=0.5s | 1С:Бухгалтерия | (2, 3) | Иконка 1С + «Выручка ₸384M» |
| t=1.5s | ИС ЭСФ | (10, 2) | Иконка ЭСФ + «1 247 счетов-фактур» |
| t=2.5s | Halyk Bank | (3, 6) | Иконка банка + «Остаток ₸94M» |
| t=3.5s | Bitrix24 | (9, 5) | Иконка Bitrix + «23 активные сделки» |
| t=4.5s | Excel «Поставщики_май.xlsx» | (1, 4) | Иконка Excel + название файла |
| t=5.5s | КГД | (11, 7) | Иконка КГД + «Налоговая нагрузка 8.2%» |
| t=6.5s | Goszakup.gov.kz | (5, 8) | Иконка + «3 активных тендера» |

**Параметры карточки:**
- Размер: 280×80 px
- Padding: 16px
- Border-radius: 8px
- Фон: тёмный (#0c0d14, основной фон сайта), border 1px полупрозрачный
- Шрифт названия источника: DM Sans 16px, белый
- Шрифт цифры/состояния: DM Sans 14px, серый

**Анимация появления:** fade-in от opacity 0 до 1, scale от 0.8 до 1.0, длительность 0.4s, easing Power2.out.

**Период t=7s–9s — состояние «хаоса»:**
- Все карточки на месте
- Каждая мигает своим ритмом (opacity 1.0 → 0.85 → 1.0, цикл 2–3 секунды, рандомный для каждой)
- Между карточками **нет связей** — это визуальный хаос

**Подпись внизу (появляется t=7.5s):**
> «Данные среднего бизнеса в Казахстане живут в десяти разных системах»

Стиль подписи: 18px, серый, центрированно.

### 4.4. АКТ 2 — Обработка (секунды 9–22)

**t=9.0s:** В центре viewport появляется узел DEXIS (диаметр 80px, изумрудный круг с тонкой светящейся обводкой). Анимация — scale 0→1 с overshoot, easing Back.out(1.7), длительность 0.6s.

**t=9.5–11.0s:** Карточки источников плавно перестраиваются в кольцо вокруг DEXIS. Каждая занимает позицию на радиусе ~280px от центра. Движение — easing Power3.inOut, длительность 1.5s.

**t=10.5–11.5s:** От каждой карточки источника к узлу DEXIS прорисовываются линии. Технически — SVG path с stroke-dasharray. Линии стартуют с stroke-dashoffset 100% и анимируются до 0%. Цвет — изумрудный с opacity 60%.

**t=11.0s — начало движения данных:** По каждой линии начинают двигаться маленькие точки (диаметр 6px, изумрудный цвет, лёгкое свечение). Они появляются от узла источника, движутся к DEXIS, исчезают по достижении. Используется MotionPath plugin GSAP. Каждые 1.5–2 секунды стартует новая точка с каждой линии (рандомно). Это создаёт ощущение «потока данных».

**Chain-of-thought (цепочка размышлений) — текстовые строки справа от узла DEXIS:**

Появляются по одной с typewriter-эффектом (печатается побуквенно). Каждая строка держится 1.5 секунды после завершения печати, затем плавно исчезает (fade-out 0.3s), и появляется следующая.

| Время | Строка |
|---|---|
| t=11.0–12.5s | «Считаю остатки по счетам.» |
| t=12.5–14.0s | «Сверяю с прошлой неделей.» |
| t=14.0–15.5s | «Нахожу отклонение в марже фармацевтического направления.» |
| t=15.5–17.0s | «Сопоставляю с динамикой закупочных цен.» |
| t=17.0–18.5s | «Определяю причину.» |
| t=18.5–20.0s | «Готовлю поручение ответственному.» |

**Параметры строк:**
- Шрифт: JetBrains Mono 14px (монопространственный, технический)
- Цвет: светло-серый (#a1a1aa)
- Скорость typewriter: 30мс на символ
- Ширина блока строк: 360px
- Позиция: справа от узла DEXIS (на desktop), под узлом (на mobile)

**В период t=15–18s — внутри узла DEXIS:** появляется мини-граф (упрощённая копия графа источников). Три-четыре узла в нём подсвечиваются ярче (1С, Excel поставщиков, КГД — те, которые имеют отношение к выявленной аномалии). Линии между ними становятся ярче. Это визуально показывает, что DEXIS «нашёл паттерн».

**Подпись внизу обновляется t=18.5s:**
> «DEXIS объединяет данные и находит то, что требует внимания»

**Микропауза t=20.0–22.0s:** Все элементы держатся на месте. Узел DEXIS пульсирует (scale 1.0 → 1.05 → 1.0, цикл 1s) — готовится к выдаче результата. Это момент драматургического напряжения перед Актом 3.

### 4.5. АКТ 3 — Действие (секунды 22–30)

**t=22.0s:** Из узла DEXIS «вырастает» карточка-вывод. Анимация — scale 0→1 с overshoot, easing Back.out(1.7), длительность 0.8s.

**Карточка-вывод (большая, центральная):**
- Размер: 560×auto px (адаптивно под содержимое)
- Фон: чуть темнее основного (#13141d), border 1px изумрудный с opacity 40%
- Padding: 24px
- Border-radius: 12px

**Содержимое карточки-вывода (с денежным акцентом):**

Структура карточки — три блока сверху вниз. Первые два блока со стандартным форматированием. Третий блок — визуальная кульминация: тонкая горизонтальная линия-разделитель, лейбл капсом и крупная цифра в изумрудном цвете.

> **Маржа по фармацевтическому направлению снизилась на 1.4 пп за май**
>
> **Причина:** рост закупочной цены у поставщиков ТОО «Альфа» (+12%) и ТОО «Бета» (+8%)
>
> ──────────────────
>
> *Потенциальное сохранение маржи:*
> **₸4 800 000**

**Визуальные параметры денежного блока:**
- Лейбл «Потенциальное сохранение маржи»: 12px, серый (--text-tertiary), uppercase, letter-spacing 0.08em
- Цифра «₸4 800 000»: 36px, weight 500, цвет изумрудный (#10b981), отдельная строка
- Разделитель сверху: 0.5px solid var(--border), margin 16px 0
- Цифра появляется с лёгкой задержкой (0.4s после остального содержимого карточки) и эффектом scale 0.95→1.0 + fade-in — это визуальный кульминационный момент Акта 3

**t=22.8–24.5s:** Карточка-вывод держится на месте. Зритель читает.

**t=24.5s:** Карточка плавно трансформируется в карточку-поручение. Технически — cross-fade: текст вывода исчезает (opacity 1→0, 0.3s), затем появляется текст поручения (opacity 0→1, 0.3s). Карточка остаётся на том же месте, тех же размеров — меняется только содержимое.

**Содержимое карточки-поручения:**

> **Поручение: Коммерческому директору**
>
> Запросить коммерческое предложение у альтернативного поставщика АО «X» на квартальный объём.
>
> *Контекст и расчёты прикреплены.*
>
> [ Подтвердить и отправить → ]

Кнопка «Подтвердить и отправить» — стилизованная под действие, изумрудный фон, белый текст. На этом этапе она не интерактивна (это часть анимации).

**t=25.5–27.5s:** Карточка-поручение держится на месте. Зритель читает.

**t=27.5s:** Карточка плавно «летит» в иконку Telegram, которая появляется в нижнем правом углу viewport. Анимация: scale 1→0.3, translateX и translateY к иконке, opacity 1→0. Длительность 1s, easing Power2.inOut.

**t=27.7s:** Иконка Telegram коротко вспыхивает — короткий pulse-effect: scale 1→1.15→1, изумрудное свечение, длительность 0.5s. Это визуально подтверждает «доставку».

**Подпись внизу обновляется t=28.5s:**
> «От разрозненных данных — к готовому действию»

**t=29.0–30.0s:** Финальная пауза. Все элементы держатся неподвижно. Внизу появляется кнопка «Посмотреть ещё раз ↻» для перезапуска анимации.

### 4.6. Поведение при перезапуске

При нажатии на «Посмотреть ещё раз»:
- Все элементы сбрасываются в начальное состояние (карточки источников исчезают, узел DEXIS исчезает, чистый экран)
- Через 0.5s начинается Акт 1 заново
- GSAP timeline.restart() с очисткой всех tween-объектов

### 4.7. Mobile-адаптация (viewport < 768px)

- Карточки источников уменьшаются до 200×72 px
- Layout становится вертикальным: карточки появляются в две колонки сверху вниз
- Узел DEXIS остаётся в центре, но смещается ниже (после карточек)
- Chain-of-thought строки появляются под узлом DEXIS, не справа
- Карточки-выводы и поручений занимают всю ширину viewport с padding 16px по бокам
- Иконка Telegram «вылетает» снизу, не справа

### 4.8. Accessibility (доступность)

- prefers-reduced-motion: если у пользователя в настройках ОС включено это правило — motion-анимация не запускается автоматически. Вместо неё показывается статичная версия с тремя последовательными карточками (источники → вывод → поручение) без анимаций
- Все текстовые элементы доступны для screen readers
- На статичной версии — текстовое описание процесса в виде маркированного списка под графом

## 8. Что должно остаться неизменным после реализации

- Структура трёх актов и тайминги
- Шесть chain-of-thought строк в исходной формулировке
